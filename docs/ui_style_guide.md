# UIスタイルガイド

## UIの意図

UIは、戦場に埋め込まれた侵入コンソールのように見せます。伝播経路やアバターを覆わず、瞬間的な戦闘判断を支える必要があります。

## レイアウト

- マッチ状態は薄いトップHUDにまとめる。
- プレイヤーステータスはコンパクトにし、左から右へ素早く読めるようにする。
- モード目標やチーム状態は画面隅に予約する。
- アリーナ上に大きな浮遊カードを置かない。
- アクティブプレイ中は装飾的な説明テキストを避ける。

## コンポーネント

### プレイヤーステータス

表示内容:

- エイリアス。
- インテグリティバー。
- 感染ノードの使用可能状態。
- ファイアウォールの使用可能状態。
- オンライン/再起動状態。

可能な限り、テキストより先にバーとアイコンで読ませます。

### 感染ノード準備状態

アイコン方針:

- 小さな不規則五角形。
- 周回する進捗アーク。
- 使用可能時は所有者色。
- 使用不可時は暗いグラファイト。

### ファイアウォール準備状態

アイコン方針:

- 斜めのバリアグリフ。
- 所有者色のフィル。
- クールダウン完了付近だけ短くちらつく。

### 目標マーカー

ホログラム括弧、細い矢印、ウェイポイントリングを使います。重いファンタジー風マーカーやカートゥーン的な点は使いません。

## モーション

- HUD遷移は80〜140ms程度で速く。
- ダメージフラッシュは鋭く短く。
- 再起動状態ではプレイヤー行にグリッチを入れても、数値は読みやすく保つ。
- 画面の広い範囲で高頻度点滅を続けない。

## タイポグラフィ

- ライセンス上問題がなければ、見出しには細身のテクノ系サンセリフを使う。
- 本文ラベルは素直で読みやすいものにする。
- 大文字表記はシステムラベルとエイリアスに限定気味に使う。
- 重要数値は戦闘中に読めるサイズ以下へ縮小しない。

## 色の意味

- シアン: プレイヤー1、クリーン信号、システム所有経路。
- マゼンタ: プレイヤー2、汚染、ブラックマーケット系システム。
- バイオレット: 不安定セクター、ゲートエネルギー。
- レッド: 直近の重大警告のみ。
- グリーン: 抽出成功やキャプチャ成功のみ。

## アクセシビリティ

- 色だけでなく、形状と移動方向も併用する。
- プレイヤーごとに異なるシルエットを用意する。
- 伝播のアクティブ状態は予兆状態より明るくする。
- シアンとマゼンタの両方のエフェクト上で、被弾範囲を読めるようにする。
